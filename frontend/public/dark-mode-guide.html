üì± Automatic Theme Detection

Use CSS media queries to automatically serve the correct logo based on user's system preference:

<!-- In your HTML --> <picture> <source srcset="/bayan-logo-dark.svg" media="(prefers-color-scheme: dark)"> <img src="/bayan-logo.svg" alt="Bayan Logo"> </picture>

Dynamic theme switching with React hooks:

import { useTheme } from 'next-themes'; import Image from 'next/image'; export function Logo() { const { theme } = useTheme(); return ( <Image src={theme === 'dark' ? '/bayan-logo-dark.svg' : '/bayan-logo.svg'} alt="Bayan Analytics" width={200} height={200} /> ); }

üé® CSS-Only Solution

If you prefer CSS without JavaScript:

.logo-light { display: block; } .logo-dark { display: none; } @media (prefers-color-scheme: dark) { .logo-light { display: none; } .logo-dark { display: block; } } <!-- HTML --> <img src="/bayan-logo.svg" alt="Bayan" class="logo-light"> <img src="/bayan-logo-dark.svg" alt="Bayan" class="logo-dark">

üîñ Dark Mode Favicon

Modern browsers support theme-aware favicons using the SVG favicon with media queries:

<!-- Light mode favicon --> <link rel="icon" href="/favicon.svg" type="image/svg+xml" media="(prefers-color-scheme: light)"> <!-- Dark mode favicon --> <link rel="icon" href="/favicon-dark.svg" type="image/svg+xml" media="(prefers-color-scheme: dark)"> <!-- Fallback --> <link rel="icon" href="/favicon.svg" type="image/svg+xml">
