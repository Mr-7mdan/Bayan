# Migration Plan: Unified Query Builder (PyPika + SQLGlot)

## Objective

Migrate the current dual-file SQL generation approach (`sqlgen.py` and `sqlgen_glot.py`, totaling ~3,700 lines) into a single, unified, maintainable pipeline using **PyPika** as the query builder and **SQLGlot** for dialect transpilation.

This eliminates error-prone string concatenation, reduces manual dialect `if/else` logic, and handles complex analytical queries (pivots, running totals) much more elegantly.

## Target Architecture

1. **PyPika (The Builder):**
   - Dynamically constructs a standard ANSI SQL query using a fluent API (`Query.from_(table).select().where()`).
   - Replaces all manual AST generation and string concatenation.
2. **SQLGlot (The Transpiler):**
   - Parses the raw standard SQL generated by PyPika.
   - Compiles it natively into the target database dialect (DuckDB, Postgres, MSSQL, MySQL).

## Migration Phases

### Phase 1: Architecture & Setup

**Goal:** Establish the new pipeline foundation alongside the existing system.

- [ ] Add `pypika` to `backend/requirements.txt` (or pyproject.toml / pipenv).
- [ ] Create a new module `backend/app/query_builder.py` (or a `query_builder/` directory).
- [ ] Implement the core `QueryPipeline` class with a `compile(pypika_query, target_dialect)` method.
- [ ] Port the JSON condition parser (`_cond_sql`) to return PyPika `Criterion` objects (e.g., mapping `{ "Field__gte": 10 }` to `Field("Field") >= 10`).

### Phase 2: Migrate `build_distinct_sql`

**Goal:** Prove the pipeline on a simpler query type (Distinct / Dropdowns).

- [ ] Replace custom date-part extraction blocks (`if dialect == 'mssql' ...`) with PyPika's `functions.Extract()`.
- [ ] Implement `build_distinct_query(source, field, where)` using PyPika.
- [ ] Route the output through SQLGlot transpilation.
- [ ] Swap usage in `backend/app/routers/query.py` (`/query/distinct` endpoints) and verify behavior.

### Phase 3: Migrate `build_sql` (The Core Engine)

**Goal:** Replace the main string-concatenation engine used for standard previews and datasets.

- [ ] Map `base_select` logic into PyPika.
- [ ] Map custom columns to PyPika `Term.wrap(expr).as_(name)` (relying on SQLGlot to handle dialect casting later).
- [ ] Map JSON `joins` arrays to PyPika `.join(Table).on(condition)`.
- [ ] Map `transforms` (filters, sorts, limits) to PyPika `.where()`, `.orderby()`, and `.limit()`.
- [ ] Ensure the return signature exactly matches the expected `(sql, colnames, warnings)` tuple used by `query.py` and `datasources.py`.

### Phase 4: Migrate Analytical Builders (`sqlgen_glot.py`)

**Goal:** Replace the complex, verbose AST manipulations with clean PyPika builder code.

- [ ] **`build_aggregation_query`:** Implement grouping and multi-level aggregation using PyPika.
- [ ] **`build_pivot_query`:** Dynamically generate PyPika `Case().when().else_()` statements for pivot columns (reducing ~150 lines of AST code to ~20 lines).
- [ ] **`build_period_totals_query`:** Implement time-bucketing and running totals using PyPika window functions (`.over()`).

### Phase 5: Refactor Routers & Cleanup Imports

**Goal:** Disconnect all endpoints from the legacy files.

- [ ] Review `backend/app/routers/query.py` and `backend/app/routers/datasources.py`.
- [ ] Replace inline imports like `from ..sqlgen import _normalize_expr_idents` with equivalents from the new `query_builder` module.
- [ ] Ensure all query execution paths route through the new unified builder.

### Phase 6: Validation & Deletion

**Goal:** Final checks and removing dead code.

- [ ] Run comprehensive tests across different database dialects (DuckDB local, attached MySQL, remote MSSQL/Postgres).
- [ ] Validate complex pivot and custom column queries.
- [ ] Delete `backend/app/sqlgen.py`.
- [ ] Delete `backend/app/sqlgen_glot.py`.

## Example: The Transformation

**Before (Current `sqlgen.py` snippet):**

```python
if d == 'mssql':
    if part == 'Year': expr = f"YEAR({col})"
    elif part == 'Month': expr = f"MONTH({col})"
elif d in {'postgres', 'duckdb'}:
    if part == 'Year': expr = f"EXTRACT(year FROM {col})"
    elif part == 'Month': expr = f"EXTRACT(month FROM {col})"
```

**After (New Pipeline):**

```python
# 1. PyPika generates standard ANSI SQL regardless of dialect
expr = fn.Extract("year", Field(col))
q = Query.from_(table).select(expr)
standard_sql = q.get_sql() 

# 2. SQLGlot automatically translates to the target dialect
final_sql = sqlglot.transpile(standard_sql, read="postgres", write=target_dialect)[0]
```
